# 什么是 InkWeave

InkWeave（墨织）是一个面向创作者与读者的**结构化叙事平台**，为长篇连载、同人衍生、互动实验型文本提供容器级支持。

## 核心理念

创作工具应该专注于创作本身。InkWeave 的设计哲学是：

- **创作者优先** - 所有功能围绕创作流程设计，从草稿到发布的完整链路
- **结构化叙事** - 作品 > 卷 > 章节的三级容器结构
- **沉浸式体验** - 写作和阅读都应该是无干扰的
- **数据安全** - 双轨草稿系统（本地 + 云端），告别丢稿焦虑

## 功能概览

### 创作端

| 功能 | 说明 |
|------|------|
| 多作品管理 | 一站式管理所有创作，支持草稿/审核/发布/归档状态流转 |
| CodeMirror 编辑器 | 基于 CodeMirror 6 的 MDX 编辑器，语法高亮、行号、活动行标记 |
| 双轨草稿系统 | 本地 IndexedDB 存储 + 云端版本快照，双保险 |
| 自动保存 | 每 30 秒自动保存到云端，编辑中实时保存到本地 |
| 卷章结构 | 作品 > 卷 > 章节三级管理，支持拖拽排序（开发中） |
| 元数据管理 | 标题、简介、封面、标签、分类、内容分级、可见性 |
| 侧边预览 | 编辑器右侧实时 HTML 预览面板 |

### 阅读端

| 功能 | 说明 |
|------|------|
| 自定义阅读 | 字体大小(14-24px)、行高(1.2-2.5)、阅读宽度(65ch) |
| 六款主题 | 默认、深夜、羊皮纸、赛博朋克、森林、海洋 |
| 进度追踪 | 自动记录每章阅读位置和进度百分比 |
| 章节目录 | 从标题结构自动生成，桌面端侧边栏 + 移动端底栏 |
| 滚动进度条 | 顶部显示当前阅读进度 |
| 自动隐藏工具栏 | 向下滚动时隐藏，向上滚动时显示 |
| 键盘导航 | 左右箭头切换章节，T 键切换主题，+/- 调节字号 |
| 作者有话说 | 章节末尾展示作者笔记 |

### 用户与权限

| 功能 | 说明 |
|------|------|
| JWT 认证 | 基于 jose 库的 JWT Token，httpOnly Cookie |
| 五级角色 | 读者、作者、编辑、管理员、超级管理员 |
| RBAC 权限 | 完整的角色-权限矩阵，覆盖作品/章节/评论/用户管理 |
| 注册/登录 | 邮箱 + 密码注册，bcrypt 加密 |

### 技术特性

| 特性 | 说明 |
|------|------|
| SSR/SSG | Next.js 16 服务端渲染，SEO 友好 |
| 类型安全 | TypeScript + Zod 全链路类型保障 |
| 响应式 | 桌面端与移动端完美适配 |
| SEO | OpenGraph 元数据 + JSON-LD Book 结构化数据 |
| CSS 变量 | 全组件使用 CSS 变量，主题切换即时生效 |

## 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Next.js 16 (App Router) |
| UI | React 19 + Tailwind CSS 4 |
| 组件库 | Radix UI |
| 编辑器 | CodeMirror 6 |
| 状态管理 | Zustand + TanStack Query |
| 数据库 | SQLite (better-sqlite3) |
| ORM | Drizzle ORM |
| 表单验证 | React Hook Form + Zod |
| Markdown | remark + rehype + Shiki |
| 动画 | Framer Motion |
| 图标 | Lucide React |
| 认证 | jose (JWT) + bcryptjs |

## 数据模型

```
用户 (User)
  |
  +-- 作品 (Work) --------+------- 标签 (Tag)
  |     |                  |         (多对多)
  |     +-- 卷 (Volume)    |
  |           |            +-- 内容警告
  |           +-- 章节 (Chapter)    (多对多)
  |                 |
  |                 +-- 草稿版本 (DraftVersion)
  |                 +-- 评论 (Comment)
  |
  +-- 收藏 (Favorite)
  +-- 阅读进度 (ReadingProgress)
  +-- 用户偏好 (UserPreference)
```
