# 章节编辑器

InkWeave 使用基于 **CodeMirror 6** 构建的 MDX 编辑器，提供专业级写作体验。

## 编辑器特性

- **CodeMirror 6 内核** - 高性能代码编辑器引擎
- **Markdown 语法高亮** - 实时语法着色
- **行号显示** - 精确定位内容位置
- **活动行标记** - 当前编辑行高亮
- **自动换行** - 长行自动软换行
- **侧边预览** - 右侧实时 HTML 预览面板
- **字数统计** - 实时显示字数和预估阅读时长
- **自动保存** - 双轨保存机制

## 双轨草稿系统

InkWeave 的核心特性之一是**双轨草稿系统**，确保创作者永不失稿：

### 本地草稿 (IndexedDB)

- 使用 Zustand persist 存储在浏览器 IndexedDB
- 编辑过程中实时保存
- 关闭浏览器后再次打开可恢复
- 存储键：按章节 ID 隔离

### 云端草稿快照

- 每 30 秒自动同步到服务端（仅内容有变化时）
- 手动点击"保存草稿快照"创建版本
- 支持版本历史查看（`/api/chapters/[id]/versions`）
- 每个快照包含：内容、标题、版本号、时间戳

### 草稿状态指示器

编辑器顶部显示当前草稿状态：

| 状态 | 说明 |
|------|------|
| 已保存至服务端 | 云端和本地内容已同步 |
| 本地草稿已保存 | 仅本地有最新更改 |
| 检测到未同步草稿 | 本地有服务端没有的更改，提示恢复 |
| 云端草稿同步中 | 正在上传到服务端 |

### 草稿恢复流程

当打开已有本地草稿的章节时：

1. 编辑器检测到本地 IndexedDB 中存在该章节的草稿
2. 弹出恢复对话框，显示草稿预览
3. 选择"恢复"加载本地草稿
4. 选择"丢弃"使用服务端版本

## 章节管理

在创作者中心的作品管理页面，每个章节支持：

| 操作 | 说明 |
|------|------|
| 创建章节 | 在指定卷下创建新章节 |
| 编辑内容 | 进入 CodeMirror 编辑器 |
| 修改状态 | draft / reviewing / published / archived |
| 删除章节 | 需确认操作 |

## 章节状态

| 状态 | 说明 | 可见性 |
|------|------|--------|
| `draft` | 草稿 | 仅作者 |
| `reviewing` | 审核中 | 仅作者 |
| `published` | 已发布 | 公开 |
| `archived` | 已归档 | 只读 |

## Markdown 扩展

除了标准 Markdown 语法，编辑器支持 GFM (GitHub Flavored Markdown)：

### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |
```

### 任务列表

```markdown
- [x] 已完成
- [ ] 未完成
```

### 代码高亮

使用 Shiki 进行服务端语法高亮，支持 100+ 种语言。

### 标题锚点

所有 h2-h6 标题自动生成锚点链接，方便目录导航和分享。

## 作者有话说

每个章节支持附带**作者笔记**（Author's Note），在章节末尾展示给读者。适合：

- 创作灵感说明
- 下章预告
- 致谢和引用
- 剧情分析

## 内容限制

| 允许的图片类型 | JPEG, PNG, GIF, WebP |

## 技术实现

编辑器核心依赖：

| 包 | 说明 |
|------|------|
| `@codemirror/state` | 编辑器状态管理 |
| `@codemirror/view` | 视图层和 DOM 渲染 |
| `@codemirror/lang-markdown` | Markdown 语言支持 |
| `@codemirror/language-data` | 多语言数据包（代码块高亮） |

MDX 处理流水线：

```
MDX 源码
  -> remark-parse (解析)
  -> remark-gfm (GFM 扩展)
  -> remark-rehype (转 HTML AST)
  -> rehype-highlight (代码高亮)
  -> rehype-slug (标题 ID)
  -> rehype-autolink-headings (标题锚点)
  -> TOC 提取插件
  -> 编译 HTML 输出
```
